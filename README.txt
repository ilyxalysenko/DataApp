(i) DataApp - приложение для управления карточками клиентов, разработанное с использованием Windows Presentation Foundation (WPF). 
Позволяет просматривать, редактировать и создавать новые карточки клиентов. Приложение взаимодействует с базой данных SQL Server. 
Расчитано на интеграцию, для использования конечным пользователем приложение требует доработки по части установки конфигурации подключения.

Системные требования:

Windows 7 и выше.
.NET Framework 4.6 или выше.

------------------------------------------------------------------------------------
Загрузка и запуск ConsoleModule:
------------------------------------------------------------------------------------
Загрузите приложение с GitHub.
Поместите файл StartConfiguration.xml в директории приложения (DataApp(New)\bin\Debug), (ConsoleModule\bin\Debug).
Запустите ConsoleModule.exe для настройки ConnectionString и пути к Cards.xml. Следуйте инструкциям в консоли.

------------------------------------------------------------------------------------
Использование
------------------------------------------------------------------------------------
Запуск приложения:
После запуска ConsoleModule и настройки параметров, данные из файла Cards.Xml загружены, приложение DataApp готово к использованию.
В случае, если не требуется загрузка полей экземпляров ClientCard из файла, в таком случае можно сразу запускать DataApp
после указания ConnectionString в файле StartConfiguration.xml (последний должен находится в папке с DataApp)



------------------------------------------------------------------------------------
Основные функции:
------------------------------------------------------------------------------------
-Просмотр данных: Основное окно отображает карточки клиентов в виде таблицы (WPF.DataGrid).
-Редактирование данных: Для редактирования выберите карточку в таблице, дважды нажмите указателем мыши на поле, которое
надо отредактировать, измените значение на необходимое. Проверьте корректность введенных данных - они
будут отправлены в базу данных, указанную в StartConfiguration.xml. Если данные введены корректно - поставьте галочку на строке
отредактированного поля (колонна справа "IsModify").
-Сохранение изменений: После внесения изменений нажмите кнопку "Сохранить", чтобы сохранить изменения в базе данных.

-Архитектура приложения:
Приложение разделено на три основных компонента: Model, ViewModel и View. Папка Model представляет модель данных,
содержит в себе классы Configuration и ClientCard.
ViewModel управляет взаимодействием между Model и View (MainWindow), View отвечает за пользовательский интерфейс.
